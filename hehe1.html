🧠 1. Cấp số nhân là gì?

Cấp số nhân là dãy số mà:

Mỗi phần tử sau = phần tử trước × một số không đổi

Số không đổi đó gọi là công bội (ký hiệu: qqq)

📌 Dạng tổng quát:

u1,u2,u3,...,unu_1, u_2, u_3, ..., u_nu1​,u2​,u3​,...,un​

Trong đó:

un=un−1⋅qu_{n} = u_{n-1} \cdot qun​=un−1​⋅q

📌 Ví dụ:

  • 2,4,8,16,322, 4, 8, 16, 322,4,8,16,32 → q=2q = 2q=2

  • 3,6,12,243, 6, 12, 243,6,12,24 → q=2q = 2q=2

  • 5,5,5,55, 5, 5, 55,5,5,5 → q=1q = 1q=1

🔢 2. Công thức quan trọng

✅ 2.1. Số hạng tổng quát

un=u1⋅qn−1u_n = u_1 \cdot q^{n-1}un​=u1​⋅qn−1

  • Muốn tìm phần tử thứ n → chỉ cần biết u1u_1u1​ và qqq

✅ 2.2. Tính công bội

q=unun−1q = \frac{u_n}{u_{n-1}}q=un−1​un​​

⚠️ 4. Các trường hợp đặc biệt

🔹 0 < q < 1

1️⃣ Vấn đề trong code hiện tại

  1. Không kiểm tra chia hết hoàn toàn

    • Bạn viết:

      C++
      int thuong1 = b / a;

      Nếu b % a != 0, thì b / a không phải là số nguyên chính xác, nhưng vẫn lưu vào thuong1. Ví dụ:

      a=2, b=5, c=10, d=20

      • b / a = 5 / 2 = 2 (nhưng thực tế không phải CSN với công bội nguyên)
        → Code sẽ báo 2 == 2? → sai.

  2. Không kiểm tra a,b,c khác 0

    • Trong cấp số nhân, không thể chia cho 0. Dù giới hạn 1 ≤ a,b,c,d ≤ 10^6 nên ở đây không thành vấn đề, nhưng tốt nhất vẫn nên kiểm tra tổng quát.

  3. Công bội nên là int không âm (nếu yêu cầu) hoặc có thể âm (q < 0 cũng hợp lệ)

    • Ví dụ: -2, 4, -8, 16 cũng là cấp số nhân với q = -2

2️⃣ Cách tư duy tốt hơn (Best Practice)

✅ Ý tưởng:

  1. Kiểm tra chia hết: b % a == 0, c % b == 0, d % c == 0

  2. Lấy công bội: q1 = b / a, q2 = c / b, q3 = d / c

  3. Nếu q1 == q2 == q3, in YES, ngược lại NO

🔹 Code chuẩn hơn:

C++
#include <iostream>
using namespace std;

int main() {
    int a, b, c, d;
    cin >> a >> b >> c >> d;

    // Kiểm tra chia hết
    if (b % a != 0 || c % b != 0 || d % c != 0) {
        cout << "NO\n";
        return 0;
    }

    int q1 = b / a;
    int q2 = c / b;
    int q3 = d / c;

    if (q1 == q2 && q2 == q3)
        cout << "YES\n";
    else
        cout << "NO\n";

    return 0;
}

✅ Ưu điểm của cách này:

  • Dễ hiểu: tách rõ kiểm tra chia hết và so sánh

  • Dễ mở rộng cho nhiều số hơn (5, 6 phần tử…)

        if(a[1] % a[0] != 0) isGP = false;
        for(int i=1;i<n;i++){
            if(a[i-1] == 0 || a[i] % a[i-1] != 0 || a[i] / a[i-1] != q){
                isGP = false;
                break;
            }
        }
    }

    cout << (isGP ? "YES\n" : "NO\n");
}
